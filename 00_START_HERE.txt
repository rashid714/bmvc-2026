CUSTOMIZATION
═══════════════════════════════════════════════════════════════════════════════

To modify training parameters, edit configs/multimodal_cloud.json:

  "epochs": 4              → Increase for better results
  "batch_size": 32         → Reduce if OOM
  "learning_rate": 2e-5    → Adjust for stability
  "max_rows_per_source": 5000  → More data = longer training
  "seeds": [41, 42, 43]    → Reproducibility
  "cloud_sources": [...]   → Which datasets to load
  "fp16": true             → Mixed precision (recommended)
  "num_workers": 4         → Data loading parallelism

═══════════════════════════════════════════════════════════════════════════════
TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

❌ CUDA Out of Memory
  → Reduce batch_size in config
  → Enable gradient accumulation: "grad_accum_steps": 2

❌ Datasets Not Found (Local Testing)
  → This is NORMAL! Datasets only available on cloud
  → Code automatically falls back to synthetic data
  → Full cloud will stream from HuggingFace Hub

❌ Distributed Training Hangs
  → Check GPU interconnect (NCCL)
  → Set: export NCCL_DEBUG=INFO
  → Try fallback: python (single-GPU mode)

❌ Training Very Slow
  → Increase batch_size (if VRAM allows)
  → Reduce num_workers
  → Use more GPUs
